# 산소킴 MVP QA 체크리스트

이 문서는 산소킴 MVP를 QR 코드로 실제 토스앱에서 검증하기 위한 출시 전 체크리스트입니다.

## 콘솔 설정값 적용

- 보상형 광고 그룹 ID는 `src/constants/sansokimRewardAds.ts`에서 교체합니다.
  - 상자 열기 기회: `SANSOKIM_BOX_OPEN_OPPORTUNITY_REWARD_AD_GROUP_ID`
  - 부스트: `SANSOKIM_BOOST_REWARD_AD_GROUP_ID`
- 프로모션 코드는 `src/constants/sansokimPromotionConfig.ts`에서 교체합니다.
  - `SANSOKIM_PROMOTION_CODE_PLACEHOLDER` 위치에 콘솔에서 발급받은 프로모션 코드를 넣습니다.
  - 테스트 지급 검증 중에는 `useTestPromotionCode: true`를 유지해 `TEST_{promotionCode}`로 호출합니다.
  - 운영 전환 시에만 `useTestPromotionCode: false`로 변경합니다.
- 프로모션 코드가 placeholder 상태이면 실제 포인트 지급 SDK가 호출되지 않아야 합니다.

## 빌드와 QR 실행

- 의존성이 설치된 상태에서 `npm run build`를 실행합니다.
- Apps in Toss 콘솔에 `.ait` 빌드 결과물을 업로드합니다.
- 콘솔 QR 코드로 토스앱에서 산소킴을 실행합니다.
- `getAnonymousKey`가 정상 동작해 동일 사용자의 로컬 상태가 복원되는지 확인합니다.

## 샌드박스와 QR 검증 구분

- 샌드박스에서 확인 가능한 항목:
  - 화면 렌더링
  - 로컬 저장 복원
  - 상자 적립 시간 계산
  - 상자 4회 탭 UX
  - 보유 상자 0개, 일일 한도 도달 등 로컬 차단
- QR 코드로 실제 토스앱에서 확인해야 하는 항목:
  - 보상형 광고 로드
  - 광고 완료와 취소 이벤트
  - `getAnonymousKey` 실제 반환
  - `TEST_{promotionCode}` 기반 토스 포인트 1원 지급
  - 지급 성공/실패에 따른 상태 차감 여부

## 보상형 광고 QA

- 테스트 광고 ID `ait-ad-test-rewarded-id`로 광고 로드가 되는지 확인합니다.
- 상자 열기 기회 받기에서 광고 완료 시에만 상자 열기 기회 1회가 생기는지 확인합니다.
- 광고 취소 또는 표시 실패 시 상자 열기 기회가 생기지 않는지 확인합니다.
- 2시간 부스트 받기에서 광고 완료 시에만 부스트가 적용되는지 확인합니다.
- 광고 취소 또는 표시 실패 시 부스트 사용 횟수와 종료 시간이 변하지 않는지 확인합니다.
- 광고 로드 실패 문구가 `광고를 불러오지 못했어요. 잠시 후 다시 시도해 주세요.`로 보이는지 확인합니다.
- 광고 미완료/닫힘 문구가 `광고 시청이 완료되지 않았어요.`로 보이는지 확인합니다.

## 토스 포인트 지급 QA

- `TEST_{promotionCode}`로 상자 1개 열기 시 토스 포인트 1원이 지급되는지 확인합니다.
- 지급 성공 후에만 보유 상자 1개와 상자 열기 기회 1회가 차감되는지 확인합니다.
- 지급 실패 시 보유 상자와 상자 열기 기회가 유지되는지 확인합니다.
- 예산 부족 계열 실패 시 `본 프로모션은 예산 소진 등으로 중단될 수 있어요.` 문구가 보이는지 확인합니다.
- 일반 실패 시 `토스 포인트 지급을 완료하지 못했어요. 잠시 후 다시 시도해 주세요.` 문구가 보이는지 확인합니다.
- 보유 상자 0개에서 상자 열기 기회가 있더라도 포인트 지급 SDK가 호출되지 않는지 확인합니다.
- 오늘 지급액이 300원에 도달하면 포인트 지급 SDK가 호출되지 않는지 확인합니다.
- 상자 열기 기회가 없으면 포인트 지급 SDK가 호출되지 않는지 확인합니다.

## 정책과 롤오버 QA

- 산소 상자가 1시간마다 1개 적립되는지 확인합니다.
- 부스트 중에는 산소 상자가 2배 속도로 적립되는지 확인합니다.
- 부스트 사용이 하루 12회를 넘으면 차단되는지 확인합니다.
- 보유 상자가 1000개를 초과하지 않는지 확인합니다.
- KST 자정이 지나면 오늘 받은 토스 포인트와 오늘 부스트 사용 횟수가 0으로 리셋되는지 확인합니다.
- 앱 종료 후 재진입해도 부스트 종료 시간과 상자 적립이 로컬 저장 상태 기준으로 이어지는지 확인합니다.

## 문구 QA

- 사용 가능한 문구만 노출되는지 확인합니다.
  - `상자 1개를 열면 토스 포인트 1원이 지급돼요`
  - `오늘은 최대 300원까지 받을 수 있어요`
  - `상자 열기 기회 받기`
  - `광고 시청 후 상자 열기 기회 1회 지급`
  - `2시간 부스트 받기`
  - `광고 시청 후 부스트 적용`
  - `본 프로모션은 예산 소진 등으로 사전 고지 없이 중단될 수 있어요`
- 금지 문구가 노출되지 않는지 확인합니다.
  - 뽑기
  - 당첨
  - 랜덤 보상
  - 대박
  - 현금
  - 출금
  - 포인트 교환
  - 광고 클릭하면 포인트 지급
